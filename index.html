Hexo와 GitHub를 이용한 블로그 생성하기

이 블로그는 Hexo를 이용해서 생성했고, GitHub Pages에서 호스팅을 해준다. 첫 번째 포스트에 블로그 생성 및 배포 과정을 담아본다.

Hexo를 선택한 이유

이 블로그는 내가 만든 두 번째 블로그이다. 몇 달 전에 Jekyll을 이용해서 블로그를 만들어 보았는데, 프론트 엔드에 대한 경험도 없고 Ruby에 대한 지식도 없다 보니, 어렵고 복잡하게 느껴졌다. 내가 사용할 줄 아는 언어로 작성된 프레임워크를 선택하는 것이 중요하다고 생각했고, javascript로 작성된 Hexo를 선택했다. Hexo는 문서화가 잘 되어있고 한글 번역도 제공한다.
정적 사이트 생성기 목록은 StaticGen에서 볼 수 있다.

블로그 포스트와 theme 관리

Hexo로 블로그를 배포하면, 생성된 정적 소스코드만 배포되므로, 포스트 원문과 theme은 따로 관리해야 한다. Git을 이용해 포스트 원문과 theme을 관리하기 위해서 다음과 같이 실행했다.

블로그 배포

GitHub Pages는 두 가지 유형의 사이트를 지원한다.

  • 개인 또는 조직 사이트
    • 저장소 이름을 <username>.github.io 으로 지정해야 한다.
    • 사이트 주소는 http(s)://<username>.github.io 이다.
    • 소스 파일은 master 브랜치에 위치해야 한다.
  • 프로젝트 사이트
    • 사이트 주소는 http(s)://<username>.github.io/<projectname> 이다.
    • 소스 파일은 다음 중 한 곳에 위치해야 한다.
      • master 브랜치
      • gh-pages 브랜치
      • master 브랜치의 docs 디렉토리

나는 이미 개인 사이트가 있었기 때문에, 프로젝트 사이트로 만들어야 했다. GitHub에서 blog 라는 이름의 저장소를 생성했고, 다음과 같이 브랜치를 사용하기로 결정했다.

  • master 브랜치: 블로그 포스트 원문과 theme 관리
  • gh-pages 브랜치: 블로그 배포

origin 저장소를 설정하고, ph-pages라는 브랜치를 생성한다.

_config.yml의 배포 관련 설정을 수정한다.

git으로 배포하기 위해서 필요한 패키지를 설치하고, 블로그를 배포한다.
